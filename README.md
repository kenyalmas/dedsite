# Dedsite

Dedsite is a personal portfolio and security-learning site with a terminal-inspired interface.

It is designed to feel like a compact workspace: resume content, project highlights, security notes, AI work, and small educational tools all live in one focused place.

## What Visitors See

- A resume-style landing page with portfolio sections.
- Project and experience highlights.
- Security and AI learning notes.
- A command-palette-inspired navigation experience.
- Lightweight security-awareness pages that explain browser and password safety concepts.

## Design Direction

The site uses a dark terminal/TUI visual style with workspace tabs, compact panels, and direct writing. The goal is to feel personal, technical, and intentional instead of like a generic portfolio template.

## Security Ethos

Security-themed pages are meant to teach, not collect.

- No credential harvesting.
- No covert collection.
- No unnecessary permission prompts.
- No submission of password samples.
- No dependency-heavy security theater.

Password-related tools are for fake or throwaway samples only. Real credentials belong in a password manager.

## Project Notes

This repository is maintained with a minimalism-first approach:

- Keep features small and explainable.
- Prefer clear code over clever code.
- Avoid unnecessary dependencies.
- Treat privacy and security as part of the design, not an afterthought.
- Keep operational details in private project notes instead of the public README.
